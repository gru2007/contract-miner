ZKGRM (ZKGRM) — resources and news

About

  • ZKGRM (ZKGRM) is a privacy-first PoW jetton on TON Blockchain, inspired by the spirit of the original GRM.
  • The idea is simple: keep the fair mining culture of GRM, but make ordinary transfers private-first through ShieldedPool and protocol flows.
  • User wallets are intentionally restricted from plain direct transfers. Approved protocol wallets keep contract-driven flows working for miner, DEX and pool logic.
  • Token metadata: name = ZKGRM, symbol = ZKGRM, decimals = 9.

Why ZKGRM exists

  • GRM proved that a TON jetton can be distributed through Proof-of-Work instead of a usual presale.
  • ZKGRM continues that idea as “PoW-first GRM, but ZK”: mine first, then move value through privacy-aware infrastructure.
  • Plain wallet-to-wallet transfers are not the product. The product is a token whose normal route is shielded transfer, DEX entry/exit, miner reward or another approved protocol flow.

How it works

  • PoW mining: miners search for a valid proof using current seed, pow_complexity and expiry parameters, then send a mine transaction.
  • Secure mining mode: the proof can be bound to the recipient address, so the reward cannot be stolen by simply replacing the recipient.
  • Transfer policy: ordinary wallets block plain outgoing transfers and text notes. Contract payload transfers remain available for protocol logic.
  • Protocol mode: selected wallets can be switched into protocol status, allowing standard TEP-74 flows for DEX, miner and pool contracts.
  • Admin controls: the minter can manage metadata, PoW minters, wallet protocol status and upgrades, but forced transfer/burn/wallet upgrade is intentionally unsupported.
  • Zero Knowledge: will be created using Resistance Dog Pools when they will be ready.
